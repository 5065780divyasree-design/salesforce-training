# Salesforce Platform Basics

## 1. What is Salesforce Platform?

Salesforce is a cloud-based CRM (Customer Relationship Management) platform used to manage customers, sales, services, and business processes.

Salesforce also allows developers and admins to build custom applications.

---

# 2. Components

## App
An App in Salesforce is a collection of tabs, objects, and features grouped together for a specific purpose.

### Examples:
- Sales App, Service App, and Student Management App

---

## Object
An Object is like a database table that stores information.

### Standard Objects:
- Account, Contact, and Opportunity

### Custom Objects:
- Student, Faculty, and Attendance

---

## Tab
A Tab is used to access features inside Salesforce.

### Examples:
- Accounts Tab

Tabs help users navigate easily.

---

# 3. Difference: Configuration vs Coding

| Configuration | Coding |
|---|---|
| Uses clicks and setup tools | Uses programming languages |
| Easy and fast | Requires coding knowledge |
| Uses Flow and Validation Rules | Uses Apex and LWC |
| Best for simple logic | Best for complex logic |
| Less maintenance | More maintenance |

# 4. System Design

## Student Management App

### Objects

#### Student Object
- Student Name
- Roll Number
- Course
- Phone Number

#### Faculty Object
- Faculty Name
- Subject
- Department

#### Attendance Object
- Student Name
- Date
- Attendance Status

---

## User Interaction

1. Admin logs into Salesforce
2. Admin adds student and faculty records
3. Faculty updates attendance
4. Students data is stored in objects
5. Users access records using tabs
6. Reports are generated for attendance and student details
